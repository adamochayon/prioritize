# Quarterly priority ranking

Use case: A product team with 12 candidate features for the coming quarter and only bandwidth for ~3. Leadership each ranks them MoSCoW-style. Forcing the "Won't have" bucket surfaces disagreement and builds alignment fast.

## Config sheet values

| Key | Value |
|-----|-------|
| `mode` | `moscow` |
| `title` | `Q3 priorities` |
| `subtitle` | `Rank your top 3 bets for the quarter` |
| `results_visibility` | `always` |
| `anonymous` | `false` |

## Items

| id | name |
|----|------|
| 1 | Mobile app (iOS/Android) |
| 2 | Single sign-on (SSO) |
| 3 | Public REST API |
| 4 | Audit log |
| 5 | Onboarding redesign |
| 6 | Native integrations v2 |
| 7 | Performance improvements |
| 8 | AI-assisted query builder |
| 9 | Bulk operations |
| 10 | Advanced search filters |
| 11 | Custom dashboard templates |
| 12 | Notification rules engine |

## How to run it

Share the URL with your leadership team on Monday morning. Ask each person to spend 10 minutes ranking items into Must/Should/Could/Won't. Results are visible immediately, so reviewers can see consensus as votes come in. Thursday morning, review the results together to build your quarterly roadmap. Use the "Won't have" bucket to align on what's explicitly out of scope.
